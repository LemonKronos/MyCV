\textcolor{chillblue}{Project: Full Pipeline Acrobatic Drone base on Latent Imitation Learning}\\
\textbullet \quad \textcolor{chillgreen}{Latent Imitation Learning:} Train AI trying to mimic professional's action, but by using latent space and grading, giving AI the chances to evol pass pro's input, beyond what human are able to do.\\
\textbullet \quad \textcolor{chillgreen}{Goals:} framework designed to act as an "AI Co-pilot," denoising erratic, suboptimal human inputs into safe, expert-level 3D trajectories.\\
\textbullet \quad \textcolor{chillgreen}{Main components:} From bottom up, use Gazebo Harmonic as the simulation and physic engine. PX4 Autopilot to control drone high frequency inner loop. MicroXRCE-DDS to bridge between PX4 and ROS2. ROS2 as the main offboard operating system, handle the sensors and communication. That let the acrobatic AI, which is a python ROS2 node run on the system, at a lower frequency.\\
\textbullet \quad \textcolor{chillgreen}{Priciple:} The input is raw humman control (or higher AI control later), telementry data (inner sensors), environment data (vision, 3D scans). Acrobatic AI generate a imagine trajectory to manuever through the obstacles, like a pro, is the expected output. That will get filter through math-base avoidant algorithm before it reach the PX4, which realize the control to motors output.
    \end{twmnotebook}

\textbullet \quad \textcolor{chillgreen}{Execution:} Pre-record pro control input and other data in Gazebo to build the dataset. The Gazebo world in which records is a procuedural generative obstacles tunnel world. AI then get trained offline on that dataset. After that, run offline test and online (realtime) test when user control is put in and receive drone moverment acrodingly.\\
\textbullet \quad \textcolor{chillgreen}{Results:} The control pipeline non-AI work as intended, with the correct low-level obstacle avoidant ouput (up to 1km autonomous fly non-crash). But when run with AI brain, it show noisy ouput and even fight back user control, lead to weird behaviour.\\
\textbullet \quad \textcolor{chillgreen}{Diagnosed:} Initial model outputs identifying short-sighted design in dataset and data flow between high and low layer. In need of a better model, trained on larger dataset.\\
\textbullet \quad \textcolor{chillgreen}{Future:}Currently executing a drastic pipeline overhaul, focusing on low-level data computing and improved handling structures to achieve dynamic acrobatic responses.\\
\textbullet \quad \textcolor{chillgreen}{Summarize:} The project show that the pipeline is possible. However, the structure on both AI and Robotic side need to be more refined.

\begin{twmnotebook}[\dimexpr 0.7\textwidth-2pt\relax]{Edge\_AI\_Greenhouse\_Project.md}[0pt][height=110pt]
    \textcolor{chillgreen}{Goals:} Train a regression model to predict the correct irrigation amount for a greenhouse, then quantize it into an edge IoT device.\\
    \textcolor{chillgreen}{Principle:} In Python, build a LightGBM regression model on the dataset, prune unimportant features. After that use TL2cgen lib to convert the model trees to C code, which then get compiled to binary and run directly on ESP32 chip. On the Embedded side, EPS32 handle sensor filtering and formating, communication, safe guards and sleep-scheduling.\\
\end{twmnotebook}%
